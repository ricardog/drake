Based on the benchmark results:

1. KDE (Kernel Density Estimation)
   - Most computationally expensive operation
   - Performance degrades significantly with dataset size
   - HIGH PRIORITY for Rust offload
   - Expected speedup: 10-50x (typical for numeric computation)

2. OLS Regression
   - Moderate computational cost
   - Scales linearly with dataset size
   - MEDIUM PRIORITY for Rust offload
   - Expected speedup: 5-20x

3. Summary Statistics (Quartiles)
   - Relatively fast, dominated by sorting
   - LOW PRIORITY for Rust offload
   - Expected speedup: 2-10x

CONCLUSION:
  Offloading math to Rust IS WORTHWHILE if:
  - Users frequently create violin plots with >1000 points
  - Users need responsive LM plots with >5000 points
  - Real-time/interactive plotting is a goal

  Priority Order: 1) KDE  2) OLS Regression  3) Summary Stats

  Implementation effort vs benefit:
  - KDE: Medium effort, HIGH benefit
  - OLS: Low effort, MEDIUM benefit
  - Summary Stats: Low effort, LOW benefit

To compare with Rust performance, implement and benchmark the same
operations in rust/src/lib.rs
